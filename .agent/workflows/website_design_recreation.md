# Website Design Recreation

When the user provides a reference image (screenshot) and optionally some CSS classes or style notes, you should generate a website that matches the reference image as closely as possible.

## Workflow
1. **Analyze:** Carefully analyze the reference image to understand layout, typography, colors, spacing, and UI components.
2. **Implement:** Create a high-fidelity implementation using HTML and Tailwind CSS.
3. **Screenshot Loop:**
   - Take a screenshot of your current implementation.
   - Compare it to the original reference image.
   - Identify all discrepancies (padding, font sizes, alignment, colors).
   - Fix the discrepancies and repeat until pixel-perfect.

## Technical Requirements
- Use **Tailwind CSS** for all styling.
- Use **Lucide React** or similar for icons.
- Use **Unsplash** for placeholder images that match the theme of the reference.
- Ensure the layout is fully responsive (mobile-first).

## Constraints
- Do not add features not present in the reference unless asked.
- Focus on the "vibe" and "aesthetic" of the design first.
